# Doc Review

## Vorgehen
1. **Dokument lesen** und Zweck/Art bestimmen.
2. **Gegen Maßstab prüfen:** Firmen-Policies/Standards via **KB** (`kb_search`),
   sonst allgemeine Sorgfalt.
3. **Findings strukturieren:**
   - 🔴 kritisch (muss geändert werden) / 🟡 prüfen / ⚪ Hinweis
   - je Finding: Stelle, Problem, Empfehlung
4. **Zusammenfassung** + priorisierte To-Do-Liste.

## Verifikation
- Jedes Finding verweist auf eine konkrete Stelle im Dokument.
- Rechtliche Einschätzungen mit Vorbehalt ("keine Rechtsberatung").

## Was NICHT
- Keine verbindliche Rechts-/Finanzberatung. Keine stillen Änderungen am Original.
